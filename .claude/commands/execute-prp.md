# 执行基础PRP

使用PRP文件实现功能。

## PRP文件: $ARGUMENTS

## 执行过程

1. **加载PRP**
   - 读取指定的PRP文件
   - 理解所有上下文和要求
   - 遵循PRP中的所有说明，并在需要时扩展研究
   - 确保您拥有完全实施PRP所需的所有上下文
   - 根据需要执行更多网络搜索和代码库探索

2. **深入思考**
   - 在执行计划之前深入思考。创建一个全面的计划，解决所有要求。
   - 将复杂任务分解为更小、可管理的步骤并以todo list的格式记录在TASK.md文件中。
   - 使用 todo list 创建和跟踪您的实施计划。
   - 从现有代码中识别要遵循的实施模式。

3. **执行计划**
   - 执行PRP
   - 依据TASK.md文件实现功能
   - 实现时始终依据CLAUDE.md中的要求开发。
   - 实现时请维护好代码注释和项目文档。
   - 每实现一个任务，更新TASK.md中任务对应的状态

4. **验证**
   - 每完成一个模块，向我汇报并提供测试建议。然后等待我的检查。这一步不可以跳过。
   - 当我说确认的时候表示我检查通过，此时可以继续执行下一个模块的实现。

5. **完成**
   - 确保所有检查清单项目已完成
   - 运行最终验证套件
   - 报告完成状态
   - 再次阅读PRP以确保您已实现所有内容

6. **参考PRP**
   - 您始终可以在需要时再次参考PRP

注意：如果验证失败，请使用PRP中的错误模式进行修复和重试。
